Double Commander — is project started by Russian programmer Alexander Koblov (Alexx2000@mail.ru), as a fork of Sexi Commander (author — Radek Cervinka, radek.cervinka@centrum.cz).

Double Commander is cross-platform (both GNU/Linux and Windows) file manager, supports WCX, WFX, WDX, WLX and DSX plugin types. Last type is own DC search plugin type. Windows version of Double Commander compatible with many plugins from Total Commander (TC), which is more than half of those which I tried. And Linux version, has no such programs at all. DC comes with installed some WCX (archiver) plugins to work with cpio, deb, rpm, bz2, rar, zip, tar, gz, tgz archives as a directories, and also search plugin (DSX) and lister videoplayer plugin for Linux (WLX), other plugins can be installed manually.

1. Introduction

At the program download page we can see several versions, what available to download. First, we must choose for operating system (Windows or Linux) and the bit (32 or 64). After that, can select the type of distribution. For Windows there are an executable file .ехе or archive .zip (which is the so-called portable version).

For Linux there are versions for most popular Linux distributions and desktops environments. You can download compiled deb packets for Debian-based distributives (e.g, Ubuntu), and rpm for Linux Mandriva. Also available portable version of the archive tar.bz2, it is the same source from which you can build a program yourself. For various DE there are versions, based on Gtk2 toolkit, and version for KDE, based on Qt. I must note, that program is not yet available from repositories.

At the first launch Double Commander automatically detect system language, and set it for interface. DC translated into 8 languages. In the future, the language can be changed in the menu "Configuration  → Options → Language". Also Double Commander determine and set system theme. It looks like this:

2. Сonfiguration files

Double Commander keeps settings in configuration files at the program directory or at the user home directory. Main files are:

  • doublecmd.ini — all the main setup program (similar to wincmd.ini)

  • default.bar — toolbar settings (like TC)

  • editor.col — editor settings

  • shortcuts.ini — keyboard layout settings

  • cmdhistory.txt — command line history

  • dirhistory.txt — directories history

  • edithistory.txt — edit files history (using F4)

  • maskhistory.txt — the history of the use of masks to search, highlight, etc.

  • doublecmd.log — log all operations

  • doublecmd.ext.example — sample file associations configuration

The structure of these files is described in detail in the topic. Almost all the settings in these files can be changed from the program GUI.

3.1. Menu bar

Menu Bar provides access to all features of Double Commander. But do not forget that the majority of actions you can perform in other ways: by using hot keys or buttons on the toolbar.

3.1.1. "Files"

This menu contained almost all that can be made over the files. Consider all of the items.

Create link (internal command cm_HardLink). On this item, Double Commander creates hard link to the file under cursor, and place it to a folder, open on another panel.

Link name — here the path to link and its name. The name is the last word after the last slash in the line.

Existing destination — object, on which link is created.

Create symlink (internal command cm_SymLink). All its like one point above, but link is symbolic.

Selecting Show file properties... (internal command cm_FileProperties) we get a window like that:

 

which indicates the file or folder name, size, write and last access time, and also owner and group. On the "Properties" tab you can change the access to the file (if you have the rights to do so). But only for a file or folder under the cursor.

Using item Edit comment... (internal command cm_cm_EditComment) you can create or change comments in UTF, ISO, KOI or ANSI encodings.

Calculate Occupied Space (internal command cm_CalculateSpace) item calculates size of selected objects in rounded units and in bytes.

Compare by Contents (internal command cm_CompareContents) shows window with two panels with the files to compare (usually text).

Compare files — starts the process of comparison files. Different lines highlighted in red by default. You can choose any files. Also there is a vertical and horizontal scrolling.

Next difference — step to next different lines.

Previous difference — step to previous different lines.

Binary mode — result is presented in binary form.

Keep scrolling — allow to synchronize scrolling.

Close — closes comparison window.

Multi Rename Tool (internal command cm_MultiRename)

Allow to rename any files by defined mask. E.g. some files with different names could be renamed into Picture 1, Picture 2, Picture 3, etc.. Select files to rename and get window (see figure above). There you can see old and new names of your files.

On the panel Mask you can set mask to rename. Following macroses are avaiable:

  • [N] — is the same name as a source file

  • [Nx] — get x-character from original name. E.g. if source file named "Picture" choice [N5] would mean that will be used only character "u" (fifth in the name)

  • [Nx:x] — get character from x to x. E.g. if source file named "Picture" choice [N1:4] would mean "Pict"

  • [C] — is counter, its parameters are listed on Counter panel.

  • You can combine the use of macros, and your own text (see figure).

Panel Find & Replace will help when you need to replace parts of the file name (e.g to change "a" to "i", so change word "race" to "rice"). Regular expressions are also available.

At the panel Counter set the parameters of counter: first number, number width (1, 01, 001, 0001, etc..), interval.

At the list File Name Style you can choose from the following:

  • No change — the new names will be same as the old one.

  • UPPERCASE — the new names will be uppercase.

  • lowercase — the new names will be only lowercase.

  • First Char Big — the new file names will begin with a capital character.

At the list Log Result you can enable logfile and specify path to it.

By selecting item Pack files (internal command cm_PackFiles) we call standard dialog to pack files to the zip, tar.gz, tgz and, possibly, rar formats.

Also we can unpack files by Extract Files (internal command cm_ExtractFiles). Same dialog is called when you copy files from the archive.

First line specifies destination directory (where the archive will be unpacked). It can be changed manually or by clicking on the button right of the line. At the line Files to unpack, you can specify mask, which files will be unpacked. The history of masks can be found in the drop-down list. Checkbox Unpack each archive to a separate subdir (name of the arcive) allows to uncompress several archives isolated, each in its folder.

Items Split file and Combine files (internal commands cm_FileSpliter and cm_FileLinker)

File source — specifies a file to split.

Directory target — need to specify destination folder of files

File size — select size of splitted parts. You can also enter your own value. Presets are:

  • 1457664B — 3.5" — Floppy drive 3.5

  • 98078KB - ZIP 100MB — ZIP drive

  • 650MB - CD 650MB — CD drive size

  • 700MB - CD 700MB — CD drive size

On click OK button you get some files with extension .split at the target directory. To combine file again, select all ".split" parts and click Combine files(cm_FileLinker). You need also specify order of slpit parts to combine.

This function is very useful, if you have small drives and large files. It is possible to move the file parts (as in the TC).

Item Calculate check sum... (internal command cm_CheckSumCalc) is used to calculate the checksum (MD5 and SHA1) to check files integrity.

Save check sum file(s) to: — line scpecifies name and path to the calculated checksum.

Create separate MD5 files for each file — if checksum calculates for multiple files, this option allow to create own MD5 file for each one.

MD5 and SHA1 — select algorithm which will calculate checksums.

There are some commands at this group with which you can select files by different ways.

First and second items Select a group and Unselect a group (internal commands cm_MarkPlus and cm_MarkMinus). This commands select group of files by mask. As mask you can specify file names or extensions.

E.g. will be selected all files that have extension .png. Also there is a drop list with masks history.

Third and fourth itemsSelect all (internal command cm_MarkMarkAll) и Unselect all (internal command cm_MarkUnmarkAll). They help to select or unselect all items from panel.

Fifth item is Invert selections (internal command cm_MarkInvert). It makes selected files unselected and mark unselected.

Sixth and seventh items Select all with same extension (internal command cm_MarkCurrentExtension) и Unselect all with same extension (internal command cm_UnmarkCurrentExtension). These functions help to select and unselect files with some extension as the file under the cursor.

Eighth and ninth itemsCopy Filename(s) to Clipboard (internal command cm_CopyNamesToClip) and Copy Filename(s) with Full Path (internal command cm_CopyFullNamesToClip). First command copies selected filelist to clipboard, second copies too, but with full file paths.

3.1.3. "Commands"

First item is Search (internal command cm_Search).

Line File mask set the mask to search.

Next line Directory specifies directory to start search, default is "/".

Search subdirectories can have:

  • current dir only

  • some levels (from 1 to 100)

  • all (unlimited depth)

This setting control search depth at the filesystem tree. Set "current dir only", we will search only in specified "Directory" (not any subdirs).

Also you can use regular expressions.

If checkbox Find text in file enabled, process will search text, what specified at line Find Data at the files. You can also replace this text to any, specified at line Replace text and enabled checkbox.

Case sensiteive — said that the capital letters should be distinguished from the line. E.g. "Fallout" and "fallout" would be different.

Find files NOT containing the text, makes process to display files, what have not text, specified at the Find Data field. Also you can set encoding here (UTF, ISO, KOI, DOS, ANSI etc..).

Use search plugin: Use Locate plugin to make search.

At the bottom of the window is a field in which search results are displayed, and a few buttons.

View — opens selected file with viewer (F3).

New search — clean result and allow to research (perhaps with new conditions).).

Go to file — closes search window and set cursor on selected file at the panel.

Feed to listbox — gives a list of files with full paths at the file panel.

Tab Advanced let to specify additional conditions: Date (from .. to), Time (from .. to), Size (from .. to), also attributes and file age.

Second item is Directory hotlist (internal command cm_DirHotList). This command same as pressing "*" button.

Selecting this item we get popup menu, at which we can choose, add, delete or configure bookmarks.

Add — opens a dialog add a new bookmark.

Delete — deletes bookmark.

Add manually — allows to enter path of new bookmark from keyboard.

Edit — allows to edit added bookmark (Settings keep at section [Configuration] of doublecmd.ini, HotDir= all bookmarks, separated by commas).

Third item is Run Term (internal command cm_RunTerm) — opens terminal at current directory (settings at Run terminal).

Fourth item is Open VFS list (virtual file system) used by plugins (WFX, WCX, etc), that needs own filesystem. If you add and configure the relevant plugins, the command will open the list, providing an opportunity to work with them.

Fifth item Swap Panels (internal command cm_Exchange), swaps left and right panels.

Sixth item is Target = Source (internal command cm_TargetEqualSource)  — opens same directory at inactive panel. This command same as pressing "=".

3.1.4. "Tabs"

This group needs to manage the tabbed file panels.

First item is New Tab (internal command cm_NewTab) — adds new tab at the panel and choose it.

Second items is Open folder in new tab (internal command cm_OpenDirInNewTab) — opens new tab for directory under cursor, but not choose it.

Seventh and eighth items are Switch to next tab (internal command cm_NextTab) and Switch to previous tab (internal command cm_PrevTab)  — to switch to the right tab (the next) and the left (previous) tab.

Drives button bar helps you to change drive quickly with one mouse click. Right click calls context menu that contain operations as mount/unmount, format, etc. (depends on OS and drive).

3.4. Drives list

Helps to change drive, also contains additional information: size, free space, navigation buttons— « / .. * ~ =».

Internal commands are cm_LeftOpenDrives for left panel and cm_RightOpenDrives — for right. Default shortcuts: (Alt+F1 and Alt+F2 but they don't work for must Linux Desktop Environments.

Additional navigation buttons:

«/» — go to file system root

«..» — go to parent directory

«*» — calls hotlist. See section Directory hotlist

«~» — go to home directory

«=» — see command Target = Source (Alt+Z by default).
