В: Что такое Double Commander?
О: Double Commander - это двухпанельный файловый менеджер. Подробности можно прочитать здесь.

В: Что нового в этой версии?
О: Информацию об истории версий вы можете найти в интернете на этой странице. Но сейчас проект развивается настолько быстро, что последние ночные сборки, возможно, содержат множество новых возможностей и улучшений.

В: Это версия Total Commander® для Linux, да?
О: Нет, это независимая программа от других разработчиков. Сейчас существуют версии для ОС Windows® и Linux с графическими интерфейсами gtk2 и qt. В будущем вполне возможно появление версий программы для других операционных систем, например Mac OS.

В: В чем отличия этой программы от моего любимого Total Commander®?
О: Цель проекта - создать программу, приближенную к TC по интерфейсу, максимально совместимую с ним по плагинам и конфигурационным файлам. При этом Double Commander обладает рядом существенных преимуществ: он абсолютно бесплатен для использования, кросплатформенный (работает под разными операционными системами), и имеет открытый исходный код.

В: У меня 64-битная ОС. Могу я скачать Double Commander, оптимизированный под мою систему?
О: Да, сборки для x64 существуют под обе поддерживаемые платформы.

В: Как можно связаться с разработчиками Double Commander?
О: Напишите в официальный форум проекта, и ваше мнение или пожелание не останется без внимания.

В: Какую версию мне скачивать?
О: Если у вас KDE - то качайте qt-сборку, иначе (Gnome, Xfce, LXDE) сборку с gtk.

В: Есть ли у проекта репозитарий?
О: Репозитория пока нет. Но вы можете скачать с официального сайта установочные пакеты deb или rpm, а также "portable" - упакованные с помощью bz или bz2 бинарники. Либо можно скачать исходный код и собрать DC самостоятельно.

В: Плагины от тотала не работают!?
О: Работают, но не все. Плагины от TC скомпилированы под винду. Поэтому их можно подключать только к Windows-версии Double Commander. Но любой плагин, имеющий открытый исходный код, можно собрать для линукс (или уговорить разработчика плагина это сделать :)).

В: Как поменять настройки цвета для файловых панелей?
О: Ответ: Чтобы изменить цвет, надо сделать следующее:

  1. Настройка->Параметры->Цвета (не получите никаких изменений, пока не создадите свой стиль колонок).
  2. Настройка->Параметры->Колонки выбираете по умолчанию (Default) нажимаете "Редактировать" и там все сами поймете :)

В: Для чего кнопки "R" и "Все" в настройках цвета?
О:

  • "R" - восстанавливает цвет на тот, который был по умолчанию.
  • "Все" - Распространяет выбранный цвет на все колонки.

В: При изменении размеров панелей колонка "Размер" налазит на колонку "Дата". Заголовок колонки изменяется нормально, а колонка ползет...
О: В настройках на странице "Поведение" установите галочку "Обрезать текст по ширине колонки".

В: Почему заголовок табулятора, при первом запуске в linux почему-то на английском?
О: При первом запуске DC определяет язык системы на основании локали. В дальнейшем текст заголовков сохраняется в doublecmd.ini и считывается из него. Изменить заголовки можно в настройках колонок.

В: Как повесить свои значки на определенные типы файлов?
О: Можно отредактировать файл конфигурации doublecmd.ext. Пример, для связи определенного значка с графическими типами файлов:

[jpeg|jpg|gif|png]
Name=images
Icon=/home/rust/progs/doublecmd/pixmaps/32x32/mimetypes/image.png
open=gthumb '%p'

При таком порядке он все перечисленные расширения будут отображаться со значком image.png.

В: ERROR: [TGtkWidgetSet.GetSysColor] Bad Value: 45. Valid Range between 0 and 31. Что это?
О: Это известная ошибка, она связана с переходом на новую версию Lazarus IDE (точнее, библиотеки LCL). Чтобы избавиться от этой ошибки, нужно в диалоге настроек на странице "Цвета"->"Файловые панели" заново выбрать цвет "Текст под курсором, а также в Настройка->Параметры->Колонки проделать то же самое.

В: Что надо сделать, чтобы при выделении пробелом, курсор сдвигался вниз на одну строчку?
О: Установить в файле doublecmd.ini в секции "Configuration" ключ SpaceMovesDown=1.

В: Есть ли здесь возможность создавать очередь копирования файлов?
О: В настоящее время нет. Будет реализовано в ближайшем будущем.

В: "Ctrl+A" не работает, где включить?
О: Дело в том, что настройки по умолчанию для хоткеев применяются в случае отсутствия shortcuts.ini (например при первом запуске после установки), так как данный файл уже существует новые настройки не применились.
Как включить:

  1. В диалоге настроек на странице "Горячие клавиши" установить данный хоткей для команды "cm_MarkMarkAll"
  2. В файл shortcuts.ini добавить следующие строки:

    3. [Ctrl+A]
    Command0=cm_MarkMarkAll
    Object0=frmMain
    Form0=frmMain

В: Скопировать папку с переименованием не получается.
О: Ситуация с переименованием директории на данный момент не обрабатывается. Возможность будет реализована позже.

В: Тут можно создавать выпадающее меню? Как это делать?
О: Можно, добавляем на панель кнопку, в качестве команды выбираем "cm_ShowButtonMenu", в параметры прописываем путь к *.bar файлу (если он находится в каталоге с конфигами, то достаточно указать только имя файла).

В: Как можно настроить контекстное меню? Например добавить пункт "Открыть с помощью"?
О: Это можно сделать в ассоциации файлов, вручную, обозначив какие расширения какими программами можно открывать, там же назначить какое приложение будет по умолчанию.
Настройка - Файловые ассоциации - типы файлов (вводите любое имя, например "media") - Расширения (добавить, например avi) - команды - добавить (действие пишите open - команда пишите, например totem-xine "%p" или mplayer "%p" или smplayer "%p" и т.д.).
После этих действий по правому клику можно выбирать чем открыть.
Кстати, можно легко сделать так, чтобы любую папку можно было открыть, например, в наутилусе (Gnome) с правами рута:
Чтобы это сделать, надо добавить в ассоциации новый тип файлов (например dir) а в качестве расширения указать folder, в качестве действия Open as root а команда: gksu nautilus "%p"
Правой кнопкой на папке, в контекстном меню выбираете Open as root и вводите пароль :)

В: Окна редактора и просмотрщика появляются где хотят, при включенном кубе рабочего стола (compiz).
О: В настройках compiz: place windows > windows with fixed positions и window rules > fixed size windows.
Код:
(class=Doublecmd) & (title=/)

В: Во встроенном редакторе странный шрифт - по горизонтали символы друг на друга наезжают.
О: Необходимо использовать моноширинные шрифты как во вьювере, так и редакторе (ограничение используемого компонента).

В: На одну единственную картинку вьювер не срабатывает. Остальные картинки просматриваются, а вот эту отказывается.
О: Судя по всему, компонент используемый для просмотра изображений не поддерживает некоторые JPG файлы. Теперь вместо ошибки такие файлы открываются как текст.

В: Стало неожиданостью... смотрелка не видит .gif :(
О: Да это так, однако компонент для отображения GIF есть, осталось только его прикрутить к просмоторщику. Будет реализовано в ближайшее время.

В: Есть такие текстовые файлы, в которых при быстром просмотре строчки уходят за край окошка! А внизу нет ползунка чтобы прочесть до конца!
О: Хм... действительно горизонтальный ползунок не предусмотрен, в данном случае можно воспользоваться режимом "В виде текста с разрывами строк".
